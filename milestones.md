# Мирослав CEO Bot — Milestones

**Старт:** 11 марта 2026

---

## M1: Скелет бота

**Цель:** бот запускается, принимает сообщения в личке, отвечает через Claude API.

**Состав:**
- Структура проекта, Dockerfile, docker-compose, `.env`, requirements.txt
- Entry point (main.py), polling
- Разделение группа/личка, фильтр по admin_id
- Обёртка Anthropic API + system prompt Мирослава
- Загрузка конфига
- Команды `/health`, `/settings`

**Критерий:** бот запущен в Docker, в личке отвечает как Мирослав, `/health` возвращает статус.

**Как выглядит приложение:**
Бот работает локально в Docker-контейнере. Админ пишет ему в личку — получает дерзкие ответы от "CEO Мирослава". Команда `/health` показывает uptime и статус API. В группе бот молчит и ничего не делает. Данных на диске нет кроме конфига.

---

## M2: Профили и команды

**Цель:** полное управление "сотрудниками" через личку.

**Состав:**
- CRUD профилей (JSON-файлы)
- Команды: `/assign`, `/backstory`, `/profile`, `/team`
- System prompt с подстановкой профилей участников
- Автосоздание профиля "Стажёр" для неизвестных

**Критерий:** профили создаются, сохраняются между перезапусками, `/team` показывает оргструктуру.

**Как выглядит приложение:**
Админ заполняет "штат" компании: назначает роли, пишет абсурдные backstory. Бот сохраняет всё в `data/profiles/`. При ответах через Claude уже использует данные профилей — знает кто CTO, кто стажёр, у кого какая история. В группе по-прежнему молчит.

---

## M3: Групповой чат

**Цель:** бот живёт в группе, отвечает по триггерам, помнит контекст.

**Состав:**
- Кольцевой буфер последних 20 сообщений
- Логика триггеров: упоминание, reply, ключевые слова, рандом ~15%
- Обработка стикеров (emoji fallback)
- Контекст для Claude: prompt + профили + buffer
- Команды: `/keywords`, `/frequency`, `/cooldown`

**Критерий:** бот в группе реагирует на триггеры, использует контекст последних сообщений, отвечает in-character, не спамит.

**Как выглядит приложение:**
Бот добавлен в группу и "слушает" чат. Упомянули @бота или написали "ceo" — Мирослав отвечает. Иногда влезает сам (~15% сообщений). Знает о чём говорили последние 20 сообщений, подкалывает конкретных людей по их профилям. Админ управляет частотой и ключевыми словами через личку.

## M4: Автоматика

**Цель:** бот живёт своей жизнью — heartbeat, обновление профилей.

**Состав:**
- APScheduler: heartbeat + batch profile update
- Heartbeat 1-2x/день (5 типов, взвешенный рандом, 09:00-21:00 Riga)
- Batch profile update раз в час (новые факты, шутки, KPI)
- Команды: `/heartbeat on/off/now`

**Критерий:** утром в группе появляется мотивационная речь CEO. Через час активного общения в профилях — новые факты и шутки.

**Как выглядит приложение:**
Бот не просто отвечает — он инициирует. Утром может выкатить "Q1 отчёт" или уволить кого-то. Вечером — мотивационную речь. Профили участников обогащаются автоматически: бот запоминает темы разговоров, внутренние шутки, генерирует фейковые KPI. Бот ощущается как живой участник чата.

## M5: Защита и админка

**Цель:** бот устойчив к ошибкам, админ контролирует всё.

**Состав:**
- Rate limit (50 вызовов/час), авто-пауза после 3 ошибок API
- Уведомления админу (80% лимита, пауза, ошибки)
- Авто-backup профилей, откат при невалидном JSON
- Команды: `/pause`, `/resume`, `/broadcast`, `/status`
- Graceful handling: фото, голосовые, длинные тексты, спецсимволы

**Критерий:** невалидный API-ключ → пауза + уведомление. Флуд → rate limit. Любой мусор на входе → бот не падает.

**Как выглядит приложение:**
Бот production-ready. Админ видит статистику токенов, может поставить на паузу, отправить broadcast от имени Мирослава. При сбоях API бот сам останавливается и сообщает админу. Профили бэкапятся каждый час. Бот переживает любой ввод без краша.

## M7: Долгосрочная память и тонкая настройка ✅

**Цель:** бот не повторяется, помнит прошлые разговоры, тон настраивается на лету.

**Состав:**
- Долгосрочная память (chat_memory.md) — саммари разговоров каждый час
- Память инжектится в system prompt при каждом вызове Claude
- Три режима тона: normal / bold / brutal, переключаемые через `/tone`
- `/help` — список всех команд
- `/updateprofiles` — ручной триггер batch update + memory
- `/probe` и `/probe @user` — допрос сотрудников для наполнения профилей
- Мирослав доверчивый — верит всему что говорят
- Batch update запоминает все заявления, даже абсурдные
- Разговорный тон без корпоративного сленга

**Критерий:** бот помнит что обсуждали вчера, не повторяет одни и те же шутки, тон можно переключить одной командой, probe заполняет профили новых участников.

**Как выглядит приложение:**
Мирослав помнит контекст прошлых разговоров — если вчера кто-то сказал что он водолаз, сегодня бот это упомянет. Админ может переключить тон с обычного на жёсткий или с матом одной командой. Новых участников можно "допросить" через `/probe` — бот сам задаст им вопросы в группе. Всё что люди говорят — запоминается, даже шутки и абсурд.
