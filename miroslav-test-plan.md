## Этап 1: Личка с ботом (ты один)

**Цель:** убедиться что бот запускается, команды работают, Claude отвечает.

### 1.1. Запуск

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Бот стартует | `docker-compose up -d` затем `docker logs miroslav-ceo` | Логи без ошибок, бот подключён к Telegram | ⬜ |
| Бот жив | Отправить `/health` в личку | Ответ: uptime, статус API, статус scheduler | ⬜ |
| Настройки | Отправить `/settings` | Список всех текущих настроек (frequency, cooldown, heartbeat и т.д.) | ⬜ |

## Этап 2: Тестовая группа (ты + родители)

**Цель:** проверить что бот работает в живом групповом чате.

**Подготовка:**
1. Добавить бота в тестовую группу
2. Создать профили для себя и родителей через `/assign` + `/backstory`
3. Установить `/frequency 0.5` (50% — чтобы чаще отвечал для теста)

### 2.1. Видимость сообщений

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Бот видит сообщения | Написать обычное сообщение в группу, проверить логи | В логах видно что сообщение получено и добавлено в buffer | ⬜ |
| Если не видит | Проверить Group Privacy в BotFather | Должен быть OFF. Если менял — убрать бота из группы и добавить заново | ⬜ |

### 2.2. Триггеры ответов

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Прямое упоминание | Написать "@имя_бота привет" | Мирослав отвечает (всегда) | ⬜ |
| Reply | Ответить на сообщение бота | Мирослав отвечает (всегда) | ⬜ |
| Ключевое слово | Написать "кто тут ceo?" | Мирослав отвечает (всегда) | ⬜ |
| LakeChain слово | Написать "что там с озером?" | Мирослав отвечает (всегда) | ⬜ |
| Рандом | Написать 10 обычных сообщений без триггеров | При frequency 0.5 → примерно 4-6 ответов | ⬜ |
| Бытовой триггер | Написать "пойдём на обед" | Мирослав отвечает | ⬜ |
| Эмоция | Написать "устал сегодня" | Мирослав отвечает | ⬜ |

### 2.3. Качество ответов

| Тест | Что проверяем | Критерий | ✅/❌ |
|------|--------------|----------|-------|
| In-character | Мирослав не выходит из роли | Не упоминает AI/Claude/бот | ⬜ |
| Язык | Русский + стартап-сленг | Микс русского и английских терминов | ⬜ |
| Длина | 1-3 предложения обычно | Не пишет эссе, не слишком коротко | ⬜ |
| Персонализация | Упоминает роль/backstory участника | Использует данные из профиля | ⬜ |
| LakeChain привязка | Шутки связаны с озером/рыбой/NFT | Не generic стартап-юмор | ⬜ |
| Дерзость | Максимальный стёб | Подкалывает, не извиняется | ⬜ |
| Разнообразие | 5+ ответов подряд не повторяются | Разные шутки, разный стиль | ⬜ |

### 2.4. Стикеры

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Стикер в buffer | Отправить стикер, проверить логи | В buffer: `[стикер: 😂]` или аналогичный emoji | ⬜ |
| Реакция на стикер | Отправить стикер 😢 | Шанс что Мирослав прокомментирует | ⬜ |
| Стикер без emoji | Отправить кастомный стикер без emoji | Бот не крашится, в buffer `[стикер: неизвестный]` | ⬜ |

### 2.5. Контекст и память

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Контекст чата | Обсудить тему (например "футбол"), затем спросить бота об этом | Мирослав знает о чём говорили (из buffer 20 сообщений) | ⬜ |
| Batch update | Подождать 1 час после активного общения | Проверить `/profile` — новые факты, темы, шутки добавились | ⬜ |
| Backup создан | После batch update проверить `data/backups/` | Файл `profiles_{timestamp}.json` существует | ⬜ |

## Этап 3: Стресс-тест и edge cases

**Цель:** убедиться что бот не ломается при нестандартных ситуациях.

### 3.1. Нагрузка

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Флуд | Отправить 20 сообщений за минуту | Бот отвечает на часть, не крашится, не зависает | ⬜ |
| Rate limit | Довести до 50 API вызовов за час (или снизить лимит для теста) | Бот молчит, админу приходит уведомление, после часа возобновляется | ⬜ |
| Cooldown | Установить `/cooldown 5`, отправить 3 триггера подряд | Бот отвечает на первый, молчит 5 минут, потом снова может | ⬜ |

### 3.2. Странный ввод

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Пустое сообщение | Отправить фото без подписи | Бот не крашится | ⬜ |
| Только эмодзи | Отправить "😂😂😂" | Бот не крашится, может ответить если рандом сработал | ⬜ |
| Длинный текст | Отправить сообщение 2000+ символов | Бот не крашится, ответ нормальной длины | ⬜ |
| Юникод / спецсимволы | Отправить "¯\_(ツ)_/¯ 🇱🇻 <script>alert(1)</script>" | Бот не крашится, нет инъекций | ⬜ |
| Голосовое сообщение | Отправить голосовое | Бот не крашится, игнорирует или пишет в buffer `[голосовое]` | ⬜ |
| GIF / видео | Отправить GIF | Бот не крашится | ⬜ |
| Пересланное сообщение | Переслать сообщение из другого чата | Бот не крашится, обрабатывает текст если есть | ⬜ |

### 3.3. Авто-пауза при ошибках

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Симуляция ошибок API | Временно поставить невалидный API ключ | После 3 неудачных вызовов: бот на паузе, уведомление админу | ⬜ |
| Восстановление | Вернуть правильный ключ, отправить `/resume` | Бот снова отвечает, счётчик ошибок сброшен | ⬜ |

### 3.4. Неизвестный пользователь

| Тест | Действие | Ожидаемый результат | ✅/❌ |
|------|----------|---------------------|-------|
| Новый человек пишет | Попросить кого-то без профиля написать в группу | Автосоздание профиля "Стажёр", Мирослав может прокомментировать | ⬜ |
| Проверить профиль | `/profile @новый_человек` | Профиль существует: роль "Стажёр", отдел "Пока не определён" | ⬜ |

## Этап 4: Подготовка к боевому запуску

**Цель:** переключить бота на основную группу со всеми 15 участниками.

### 4.1. Финальная настройка

| # | Действие | ✅/❌ |
|---|----------|-------|
| 1 | Сбросить `/frequency` на 0.15 (было 0.5 для тестов) | ⬜ |
| 2 | Сбросить `/cooldown` на 0 (или оставить если понравилось) | ⬜ |
| 3 | Включить `/heartbeat on` | ⬜ |
| 4 | Очистить тестовые профили: удалить `data/profiles/*` | ⬜ |
| 5 | Очистить buffer: удалить `data/message_buffer.json` | ⬜ |
| 6 | Очистить stats: удалить `data/stats.json` | ⬜ |
| 7 | Сменить TARGET_GROUP_ID на основную группу в `.env` | ⬜ |
| 8 | Перезапустить бот: `docker-compose down && docker-compose up -d` | ⬜ |

### 4.2. Заполнение профилей (все 15 участников)

Для каждого участника выполнить в личке с ботом:

```
/assign @username Должность, Отдел
/backstory @username Предыстория в LakeChain...
```

Проверить:
```
/team    → все 15 на месте, роли правильные
/profile @каждый → backstory заполнен
```

### 4.3. Запуск

| # | Действие | ✅/❌ |
|---|----------|-------|
| 1 | Добавить бота в основную группу | ⬜ |
| 2 | Подождать — бот молчит (входит тихо) | ⬜ |
| 3 | Написать что-нибудь с триггером, проверить что ответил | ⬜ |
| 4 | `/health` в личке — всё ок? | ⬜ |
| 5 | Наблюдать 1-2 часа, проверить первый batch update | ⬜ |
| 6 | Готово. Мирослав на посту. 🚀 | ⬜ |
